# 0022 — R3F scene + deepslate block-geometry adapter

**Milestone:** M5
**Depends on:** #0003

## Goal
Wire the R3F `<Canvas>` to a working block-rendering pipeline. We do **not** use deepslate's `StructureRenderer` — instead we use its lower-level utilities to load MC models and build a texture atlas, then expose a small adapter that turns a `(blockId, properties)` into an R3F-renderable mesh (geometry + material). This keeps everything declarative and lets overlays/highlights live in the same scene graph.

## Outcome
- Asset bundle (`assets/minecraft/{textures,models,blockstates}`) bundled as `frontend/public/mc-assets.zip`, fetched once at runtime.
- A `BlockAdapter` module: given an MC block id + properties, returns `{ geometry: BufferGeometry, material: Material, transform: Matrix4 }` suitable for an R3F `<mesh>`.
- A test harness page (`/__debug/blocks`) that renders a grid of representative blocks (stone, oak_stairs[facing=east], lever[powered=true], glass, redstone_wire, fence) to prove the adapter works.
- One shared atlas material so all blocks share a draw-call-friendly material.

## Implementation notes
- Use deepslate's `BlockDefinition`, `BlockModel`, `TextureAtlas`, `BlockStateRegistry` to parse models from the asset zip.
- Cache: `useMemo` per `(id, propsKey)` — same block state should reuse geometry.
- Performance: many tests have hundreds of blocks. Plan to batch identical block states into `<instancedMesh>` (deferred to #0023; the adapter exposes the geometry/material so instancing is straightforward).
- License note for the asset zip: see #0034 README.
- Asset prep script `frontend/scripts/fetch-assets.ts` downloads vanilla 1.21.x client jar, extracts only the relevant subtrees, zips into `public/mc-assets.zip`. Run once locally.

## Status (this issue)

Implemented at:

- `frontend/scripts/fetch-assets.ts` — Node script (run via `npm run assets`, which uses `tsx`). Reads `MC_VERSION` env (default `26.1.2`), pulls Mojang's `version_manifest_v2.json`, downloads the matching `client.jar` (~36 MB for 26.1.2), verifies SHA-1, and re-zips just `assets/minecraft/{blockstates,models,textures/block,textures/item}/...` into `frontend/public/mc-assets.zip` (~2.2 MB out, 6,852 files: 1171 blockstates, 2393 block models, 1215 block textures, 1285 item models, 793 item textures, plus a few `models/` parents). Uses `JSZip` for in-memory unzip/rezip; no `unzip`/`zip` binaries required.
- `frontend/public/mc-assets.zip` — generated artifact, gitignored. README has a "Minecraft assets" section documenting the workflow and license caveat.
- `frontend/src/world/atlas.ts` — singleton `loadBlockProviders()` (cached promise). Fetches `/mc-assets.zip`, walks the jar with JSZip, and builds three Maps keyed by `Identifier.toString()`: `BlockModel`s (after a single `flatten(provider)` pass to resolve parent chains), `BlockDefinition`s, and texture `Blob`s. Then calls `TextureAtlas.fromBlobs(...)` (deepslate, browser-only — uses `document.createElement('canvas')` + `createImageBitmap`). Atlas image becomes a `THREE.CanvasTexture` with `NearestFilter` and `SRGBColorSpace`. Returns `{ blockModels, blockDefinitions, atlas, atlasTexture, atlasSize }`. **Only `textures/block/*` go into the atlas** — item icons are out-of-scope here and slated for #0030's `ItemRenderer` pipeline.
- `frontend/src/world/blockAdapter.ts` — `buildBlockMesh(blockId, props, providers)` is the public surface. Calls deepslate's `BlockDefinition.getMesh(...)` with `Cull.none()` (per-block culling can't be computed without neighbours; #0023 should swap to a real cull mask once it groups blocks). Converts the resulting deepslate `Mesh` (quads with `Vertex { pos: Vector, color: [r,g,b], texture: [u,v] | undefined, normal: Vector | undefined }`) into a `THREE.BufferGeometry` with `position`, `normal`, `uv`, `color`, indexed (4 verts → 2 tris per quad). UVs are flipped on V because deepslate uses MC's top-left convention; positions are scaled by `1/16` (MC units → 1-block units). Skips quads with no vertices (returns `null`). One **shared** `MeshStandardMaterial` (`vertexColors: true`, `alphaTest: 0.1`, `side: DoubleSide`) is reused across all blocks via `getSharedMaterial(providers)`.
- `frontend/src/world/__debug__/BlockGallery.tsx` — gated by `?debug=blocks` in `App.tsx` (no router, per the handoff). Renders six representative blocks in a grid (`stone`, `oak_stairs[facing=east]`, `lever[powered=true]`, `glass`, `redstone_wire`, `oak_fence`) and an `OrbitControls` camera. Surfaces atlas-load errors as a red diagnostic pane.
- `frontend/src/world/CanvasShell.tsx` — smoke-test cube removed; now just lights + `OrbitControls` + a dark background. The actual world rendering lands in #0023.

Notes for downstream:

- `loadBlockProviders()` is the only entry point — call it once and pass providers down. Re-calling is cheap (cached promise). `resetBlockProviders()` exists but is for tests, not user-facing reload.
- The shared material has `vertexColors: true` so per-quad tints from deepslate work. Tinting (grass green, foliage, redstone-wire power) is currently passive — deepslate's `BlockColors` API exists but wasn't wired in. Grass/leaves will render as gray; fix in a follow-up if it bothers anyone.
- `Cull.none()` means **every face renders for every block**, including faces that would be hidden between adjacent solid blocks. Acceptable for the debug gallery (sparse blocks) but #0023 must group neighbours and pass a real `Cull` mask, or instance count will explode.
- Geometry is in 1-unit-per-block scene-space. Per-block translation is the caller's job; #0023 will do it via `<instancedMesh>` per `(id, propsKey)` group.
- `BlockMesh.transform` is currently always identity — left in the return type because #0023 may want a per-block matrix path before instancing lands. Drop or repurpose freely.
